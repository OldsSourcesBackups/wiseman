Quick Start Guide

This guide describes the basic steps for getting dependencies required by wiseman, downloading wiseman, compiling it, and completing your first successful run of wiseman. As wiseman is in a development state at the moment, these instructions will include some necessary build steps and manual movements of files. Over time, these will be simplified and changed.

Dependencies

Downloading and installing the proper versions of dependencies is critical for running the wiseman project. Backlevel dependencies, in particular, can cause difficult to debug problems. Often, “newer” versions of the dependencies will be fine but we strongly recommend that you stay with the recommended software dependencies listed in this section.

Dependency

Description

Java SE 5+ or above

This is the Java Standard Edition 1.5 available from Sun Microsystems. Be sure you have the SDK, not just the JRE. Throughout this document, I will refer to Java's Home directory as /usr/java.

JAXB 2.0

Java Architecture for XML Binding (JAXB).

  • Go to the downloads link and be sure to download the 2.0 FINAL version, though 2.0.1 may work. Note: I originally downloaded an older version and had all kinds of runtime errors. Also note: there are several options for downloading files, get the JAXB_RI2.0_FCS.

  • Once its downloaded, go to the directory where you stored it and run "java -jar JAXB2_20060426.jar" (substitute the .jar file name as appropriate).

  • Remember where it unpacked everything, for me, the "root" directory of JAXB 2.0 is: /home/pm141145/downloads/wiseman_dependencies/jaxb-ri-20060426, I will use this directory throughout the rest of the document

SAAJ 1.3

SOAP with Attachments API for Java (SAAJ).

  • Download the 1.3 version of SAAJ from the website, a .zip file should be located in your downloads directory afterwards

  • Unzip the file, you should be left with an API and an Implementation .jar file, remember where these are, we will use them later.

Tomcat 5.5.17 or above

You may already have Tomcat installed, you can use your existing installation in this case. Otherwise, download and install an appropriate distribution of Tomcat for your platform. Record the HOME directory of your Tomcat installation once it is ready, mine is: /home/pm141145/downloads/wiseman_dependencies/apache-tomcat-5.5.17

Be sure to test your Tomcat installation by going to http://localhost:8080 and looking at your Tomcat homepage.

JUnit 3.8.2

JUnit is required if you are going to compile and run the tests. JUnit 3.8.2 (NOT the latest) should work well. The build.properties file depends on ant tasks defined in the junit.jar that are not present in JUnit 4.x.

Download wiseman

For now, wiseman is available in source form only. As a result, check out the source from the CVS tree using the instructions located on the project site.

Configure

There are several configuration steps necessary to get everything ready to build and deploy into your running Tomcat instance.

  • Move JAXB libraries from open source installation to the wiseman/se/lib/jaxb directory (for me, this is "cp /home/pm141145/downloads/wiseman_dependencies/jaxb-ri-20060426/lib/*.jar se/lib/jaxb/")
  • Move the SAAJ libraries libraries to the wiseman/se/lib/jaxrpc directory (for me, this is "cp /home/pm141145/downloads/wiseman_dependencies/saaj*.jar se/lib/jaxrpc/")
  • Copy the Servlet API jar from your favorite servlet container to the root library directory. For Tomcat, we are using servlet-api.jar from the ./apache-tomcat-5.5.17/common/lib/ directory. (for me, the copy command is cp ~/downloads/wiseman_dependencies/apache-tomcat-5.5.17/common/lib/servlet-api.jar se/lib/)
  • Copy the junit.jar file from your JUnit download to the wiseman/se/lib directory. Also copy the junit.jar file to the wiseman/se/test/lib directory.
  • Edit the wiseman/se/project.properties file, ensure the properties are set for your configuration, also:
    • If you do not have an environment variable for the location of your app server, add one here. For Tomcat, this is the "catalina.home" variable. I added an entry: catalina.home=/home/pm141145/downloads/wiseman_dependencies/apache-tomcat-5.5.17
  • Edit the tomcat-users.xml file and add the following entries (don't add if they already exist): 
    <role rolename="manager"/>
<role rolename="wsman"/>
<role rolename="admin"/>
<user username="wsman" password="secret" roles="wsman"//>
<user username="admin" password="adminadmin" roles="admin,manager"/>

Compile and Run

Run the following ant targets from the wiseman/se directory (make sure your tomcat instance is running):

  • ant clean
  • ant
  • ant deploy
  • ant test

If the test target runs properly, you have successfully completed installation.
